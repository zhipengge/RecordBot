# 记友（RecordBot）

一款面向 **iPhone / iPad** 的本地优先打卡与健康指标记录应用。数据默认保存在设备本地，支持任务提醒、多用户指标对比、趋势图与数据导入导出。

---

## 功能概览

| 模块 | 说明 |
|------|------|
| **概览** | 今日打卡、连续天数、本周热力等汇总 |
| **打卡任务** | 标题/描述/分类/优先级；起止**日期**、每日**时段**；重复规则（每天 / 按周 / 工作日 / 周末）；附件；本地通知提醒 |
| **健康指标** | 预设与自定义指标；按用户记录数值；**趋势图**（多用户分曲线）；时间范围筛选 |
| **用户管理** | 多用户档案（用于区分指标归属）；默认用户；删除与级联清理 |
| **设置** | 数据导出/导入（JSON）、清除全部数据、通知权限、关于信息 |

## 技术说明

### 技术栈

- **语言**：Swift 5  
- **UI**：SwiftUI  
- **持久化**：SwiftData（本地数据库，默认非内存模式）  
- **图表**：Swift Charts  
- **最低系统**：iOS 17.0+  
- **设备**：iPhone、iPad（支持横竖屏，由工程 `UISupportedInterfaceOrientations` 配置）

### 架构要点

- **单 Target 应用**，源码按职责分目录：`Models` / `Views` / `Components` / `Utils`  
- **数据模型**（SwiftData `@Model`）：`AppUser`、`CheckInTask`、`TaskRecord`、`TaskAttachment`、`MetricType`、`MetricRecord`  
- **通知**：`UserNotifications`，任务保存时调度提醒（具体策略见 `NotificationManager`）  
- **导入导出**：`ExportManager` 将上述实体序列化为 JSON（含版本字段），导入时按 DTO 重建并写回 `ModelContext`

### 隐私与合规（产品层面）

- 核心数据**本地存储**，不依赖应用自带云同步（若未来增加云能力需单独说明与授权）  
- App Store **出口合规**：在工程中通过 `ITSAppUsesNonExemptEncryption` 声明是否使用非豁免加密（以当前 `project.pbxproj` / Info 为准）  
- 面向用户的隐私说明可参考仓库或发布物料中的《隐私政策》文本

## 使用说明（对外展示）

### 首次使用建议流程

1. 在 **设置 → 用户管理** 中添加成员（如家人），并可指定 **默认用户**（录入指标时默认选中）。  
2. 在 **指标** 页通过「预设」或「自定义」添加指标类型（如体重、身高等）。  
3. 在 **打卡** 页新建任务，设置日期范围、每日时段与重复方式，按需开启提醒。  
4. 在指标详情页 **添加记录**，选择用户后在 **趋势图** 中对比曲线（「全部用户」下按人分线显示）。

### 数据备份与迁移

1. 打开 **设置 → 数据管理 → 导出数据**，通过系统分享保存 JSON 备份。  
2. 换机或重装后可用 **导入数据** 选择该 JSON（注意：导入逻辑可能与当前版本 schema 一致，跨大版本升级时请优先阅读更新说明）。  
3. **清除所有数据** 会删除本地模型数据并移除已调度的本地通知，操作不可撤销，请先导出备份。

### 权限说明

- **通知**：用于任务提醒；可在系统设置中关闭。  
- **相册 / 文件**：仅在您主动为任务添加附件时使用。
